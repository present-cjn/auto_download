# 开发计划

## Phase 1：稳定下图工具

目标：先解决客户当前最重要的问题。

交付：

- 读取 `BT-6月1日订单总.xlsx`。
- 根据订单号创建目录。
- 下载 `Design Link` 图片。
- 支持 dry-run。
- 支持重复运行不覆盖。
- 支持小批量 smoke test。

已完成：

- `download_orders.py`
- `requirements.txt`
- `.venv` 本地依赖管理
- 初版设计文档

下一步建议：

- 增加 `download_report.csv`。
- 增加 `failed.csv`。
- 增加失败重试入口。

## Phase 2：导入规则和表格适配

目标：让系统适应更多现有工作表。

交付：

- 字段映射配置。
- 多表模板支持。
- 导入批次记录。
- 原始行保存。
- 更新策略配置。

## Phase 3：物流单号

目标：从现有 `单号` / `转单号` 过渡到系统化物流单。

交付：

- 读取主表 `单号`。
- 读取 `Sheet4` 转单号。
- 建立 `FulfillmentGroup` 和 `Shipment`。
- 支持用户选择物流公司。
- 预留物流 API 接入。

## Phase 4：数据库化

目标：从本地文件处理升级为可查询、可追溯的数据服务。

交付：

- PostgreSQL schema。
- 导入服务。
- 下载任务记录。
- 订单和明细查询。
- 失败任务重试。

## Phase 5：Web 管理界面

目标：让用户通过浏览器操作。

交付：

- 登录。
- 订单列表。
- 订单详情。
- 导入记录。
- 下载状态。
- 物流单号状态。
- 基础导出。

## Phase 6：物流跟踪和生产进度

目标：覆盖更完整的订单履约生命周期。

交付：

- 物流公司 API。
- 批量创建物流单。
- 定时物流查询。
- 物流异常提醒。
- 生产进度事件。
- 供应商视图占位。

## Phase 7：云端稳定化

目标：作为长期云服务运行。

交付：

- 权限系统。
- 对象存储。
- 后台 worker。
- 定时任务。
- 日志监控。
- 数据库备份。
- 审计日志。

## 当前优先级

```text
P0：下图工具稳定
P1：下载报告和失败重试
P2：导入导出字段映射
P3：物流单号读取/生成
P4：数据库和 Web
P5：云端稳定化
```
