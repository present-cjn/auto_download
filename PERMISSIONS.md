# 权限设计文档

当前 CLI 阶段默认单人本地使用，不实现权限系统。云端服务阶段需要引入角色权限。

## 1. 角色

| 角色 | 说明 |
|---|---|
| `Admin` | 管理员，管理用户、配置、导入导出、物流公司 |
| `Operator` | 订单操作员，处理导入、下图、物流单号、导出 |
| `Supplier` | 供应商，只查看分配给自己的生产任务 |
| `Viewer` | 只读查看人员 |

## 2. 权限矩阵占位

| 操作 | Admin | Operator | Supplier | Viewer |
|---|---|---|---|---|
| 上传订单表 | 是 | 是 | 否 | 否 |
| 下载设计图 | 是 | 是 | 否 | 否 |
| 查看订单 | 是 | 是 | 仅自己的任务 | 是 |
| 修改订单明细 | 是 | 是 | 否 | 否 |
| 创建物流单 | 是 | 是 | 否 | 否 |
| 导出表格 | 是 | 是 | 否 | 视配置 |
| 管理物流公司 | 是 | 否 | 否 | 否 |
| 管理用户 | 是 | 否 | 否 | 否 |

## 3. 敏感数据

以下字段需要限制查看和导出：

- 收件人姓名。
- 地址。
- 电话。
- 邮箱。
- 物流 API 密钥。

## 4. 待确认问题

- 供应商是否需要登录系统。
- 是否需要按供应商隔离订单附件。
- 是否需要给客户开放只读查询。
- 是否需要操作审批。
