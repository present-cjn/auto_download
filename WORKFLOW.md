# 业务流程文档

## 1. 当前人工流程

当前已知流程：

1. 客户每天提供订单 Excel。
2. 操作人员根据订单号和 `Design Link` 手动下载设计图。
3. 操作人员使用现有表格维护物流单号或转单号。
4. 操作人员按需要导出或整理其他工作表。
5. 后续可能人工跟踪物流和生产进度。

当前关键痛点：

- 下图工作重复、耗时、容易漏。
- 同一个订单可能有多个商品和多个设计链接。
- 同一个订单可能共用一个物流单，也可能拆成多个物流单。
- 表格字段和后续业务动作没有统一数据结构支撑。

## 2. 当前工具流程

```text
订单 Excel
  -> 解析工作表 12
  -> 读取订单号和 Design Link
  -> 按订单号创建 orders/<订单号>/
  -> 下载 Google Drive 图片
  -> 保留成功/失败结果
```

当前默认规则：

- 一行 Excel 主表数据视为一个订单明细。
- 同一订单号多个 `Design Link` 全部下载。
- 输出目录按客户订单号组织。
- 下载功能优先于完整订单系统。

## 3. 目标系统流程

```text
上传每日订单表
  -> 创建导入批次
  -> 解析原始行
  -> upsert 客户订单
  -> 创建订单明细
  -> 创建设计图 Asset
  -> 自动或手动生成履约分组
  -> 下载设计图
  -> 获取/生成物流单号
  -> 导出客户需要的表格
  -> 跟踪物流和生产进度
```

## 4. 异常处理原则

- 单个订单失败不影响整个批次继续处理。
- 所有导入行保留原始数据。
- 下载失败记录错误原因和原始链接。
- 物流单号生成失败时保留请求参数和接口返回。
- 人工修改过的关键字段后续导入不能随意覆盖。

## 5. 待确认流程

- 每天具体由谁上传表格。
- 下载后的图片如何交付给供应商。
- 当前还有哪些导出表，以及每张表的字段顺序。
- 物流单号由谁触发生成，是否需要人工审核。
- 生产进度是由内部人员维护还是供应商维护。
